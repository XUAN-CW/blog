给wsl2的ubuntu分配固定IP，优化windows开发环境

Paco
首席翻车程序验尸官

我们知道wsl2是基于hyper-v的虚拟机，每次重新启动的时候，都会重新拉一个新的hyper-v虚拟机实例，然后虚拟网卡的IP是dhcp随机分配的，如果作为开发系统用，就会比较烦每次都要换一个IP。

之前我是利用火绒在系统hosts文件里面维护一个主机名翻译，然后所有访问的wsl2的都访问这个主机名，这样，每次开wsl2只需要一步设置就可以找到了。但是这样仍然不够优雅。

今天搜索了一下，有人提供了个脚本给了我思路，他写了个bat脚本在windows里面把wsl拉起来，同时用root权限给wsl实例分配了一个固定的IP，然后在windows宿主分配了一个同网段的IP，实现了固定IP访问wsl。 这个脚本我研究了一下，可以顺便把wsl拉起来的时候需要启动的服务，比如ssh，docker等拉起来，这样同时也解决了wsl不支持systemd服务自启动的麻烦，说干就干，上脚本。

@echo off
setlocal enabledelayedexpansion

::不管三七二十一先停掉可能在跑的wsl实例
wsl --shutdown ubuntu
::重新拉起来，并且用root的身份，启动ssh服务和docker服务
wsl -u root service ssh start
wsl -u root service docker start | findstr "Starting Docker" > nul
if !errorlevel! equ 0 (
    echo docker start success
    :: 看看我要的IP在不在
    wsl -u root ip addr | findstr "192.168.120.181" > nul
    if !errorlevel! equ 0 (
        echo wsl ip has set
    ) else (
        ::不在的话给安排上
        wsl -u root ip addr add 192.168.120.181/24 broadcast 192.168.120.0 dev eth0 label eth0:1
        echo set wsl ip success: 192.168.120.181
    )


    ::windows作为wsl的宿主，在wsl的固定IP的同一网段也给安排另外一个IP
    ipconfig | findstr "192.168.120.100" > nul
    if !errorlevel! equ 0 (
        echo windows ip has set
    ) else (
        netsh interface ip add address "vEthernet (WSL)" 192.168.120.100 255.255.255.0
        echo set windows ip success: 192.168.120.100
    )
)
pause


脚本比较直白，主要步骤看注释，windows bat脚本我也不熟，就是照瓢画葫芦。这个脚本需要管理员权限运行，所以需要右键，然后“以管理员身份运行”，执行完效果如下。

好了，mobax尝试连上了。

不过还是不够方便，还要右键选”以管理员身份运行“，不够偷懒的程序员不是合格的程序员，要让这个脚本默认以管理员身份运行。

先右键这个bat脚本，给它建个桌面快捷方式

然后，右键选中快捷方式，选属性-》快捷方式-》高级，弹出高级属性窗口里面，选择”用管理员身份运行“

大功告成，以后直接运行这个快捷方式，就会以管理员身份运行我们刚才写的”wsl2-docker.bat"了。

有人问了，既然windows里面有power shell或者terminal可以打开wsl窗口了，为啥要在wsl里面跑ssh？回答是这样可以用很多更强大或者好看的虚拟终端工具，比如SecureCRT，Termius，MobaX。

还有一个更重要的杀手锏，类似MobaX这样可以运行一个linux原生的X11图形化界面，这样的话，可以把我喜欢的Jetbrains全家桶装在wsl2 linux里面，得到非常接近linux原生的IDE开发界面，具体做法见我前面的文章。
